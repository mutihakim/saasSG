# 03 Features - i18n (EN/ID)

## Tujuan

Dokumen ini menjadi acuan baku i18n untuk `project` agar konsisten end-to-end:

- Frontend React/Inertia: semua label, judul, breadcrumb, tombol, status, toast.
- Backend API envelope: `error.message` + `error.details.hint` selalu mengikuti locale aktif (`en|id`).
- Error pages web (403/404/500/upgrade/forbidden/access-required) selaras bahasa.

## Prinsip Wajib

1. Locale resmi hanya `en` dan `id`.
2. Tidak boleh hardcoded string user-facing di komponen/runtime aktif.
3. Semua error API berbasis `error.code`, lalu dipetakan ke katalog terjemahan backend.
4. Toast error wajib 2 baris:
   - baris 1: judul singkat (dari Frontend translate via `api.error.<CODE>.title`)
   - baris 2: penjelasan manusiawi (dari Backend validation message atau `api.error.<CODE>.detail`)
5. Locale harus konsisten frontend-backend dalam request yang sama.

## Struktur File (Frontend)

Gunakan struktur modular, bukan satu file raksasa:

- `resources/js/locales/en/common.json` — key umum dan api.error.*
- `resources/js/locales/en/layout.json`
- `resources/js/locales/en/auth.json`
- `resources/js/locales/en/admin.json`
- `resources/js/locales/en/master.json` — semua modul Master Data
- `resources/js/locales/en/tenant/{shared,members,roles,invitations,whatsapp,dashboard,errors,settings}.json`
- struktur `id` identik.

Aturan namespace:

- `tenant.members.*`
- `tenant.roles.*`
- `tenant.invitations.*`
- `tenant.whatsapp.*`
- `tenant.dashboard.*`
- `tenant.errors.*`
- `tenant.settings.*`
- `master.uom.*`, `master.currencies.*`, `master.tags.*`, `master.categories.*`
- `layout.shell.*`
- `api.error.*` (di `common.json`)

## Struktur File (Backend)

Katalog message backend:

- `lang/en/api.php` — custom error codes dan hints (dalam Bahasa Inggris)
- `lang/id/api.php` — custom error codes dan hints (dalam Bahasa Indonesia)
- `lang/en/validation.php` — pesan validasi standar Laravel (Bahasa Inggris)
- `lang/id/validation.php` — pesan validasi standar Laravel (Bahasa Indonesia)

Format `api.php`:

```php
return [
  'codes' => [
    'FEATURE_NOT_AVAILABLE' => [
      'message' => '...',
      'hint' => '...'
    ],
  ],
];
```

`ApiResponder` menjadi source-of-truth untuk menerjemahkan `error.code` menjadi `message`/`hint`.

## Alur Locale End-to-End

```
User klik Topbar Switch (EN/ID)
   ↓
i18next.changeLanguage() → localStorage:I18N_LANGUAGE
   ↓
Axios interceptor kirim header:
  - X-Locale: en|id
  - Accept-Language: en|id
   ↓
Backend: SetRequestLocale middleware (Global) → App::setLocale(en|id)
   ↓
Backend: EnsureTenantAccess middleware
  → Cek X-Locale header
  → Jika ada X-Locale: SKIP tenant.locale override (pilihan user menang)
  → Jika tidak ada X-Locale: gunakan tenant.locale dari database
   ↓
API validation error → teks dari lang/{locale}/validation.php
API custom error   → teks dari lang/{locale}/api.php via ApiResponder
   ↓
Frontend: parseApiError() di apiError.ts
  → Baca error.code dari response
  → Translate judul via i18n: t('api.error.<CODE>.title')
  → Detail tetap dari server (sudah dalam bahasa yang benar)
   ↓
notify.error({ title: translatedTitle, detail: serverDetail })
```

### Prioritas Locale Backend

| Prioritas | Sumber | Keterangan |
|---|---|---|
| 1 | `X-Locale` header | Pilihan user dari Topbar — tidak bisa ditimpa |
| 2 | `Accept-Language` header | Fallback bahasa browser |
| 3 | `tenant.locale` DB | Default bahasa organisasi (hanya jika tidak ada X-Locale) |
| 4 | `config/app.locale` | Default global (`en`) |

## Kontrak Error API

Semua endpoint harus tetap:

```json
{
  "ok": false,
  "error": {
    "code": "FEATURE_NOT_AVAILABLE",
    "message": "...localized...",
    "details": {
      "hint": "...localized..."
    }
  }
}
```

Standard Laravel 422 Validation Error:

```json
{
  "message": "...localized...",
  "errors": {
    "code": ["The code has already been taken."]
  }
}
```

Catatan:

- `code` stabil (machine-readable).
- `message/hint` localized (human-readable, ikuti locale aktif).
- kode teknis boleh tampil di network log, tidak wajib ditampilkan mentah di toast.

## Cara parseApiError Bekerja

**File:** `resources/js/common/apiError.ts`

```
1. Baca error.code dari response
   → Cari terjemahan di Frontend: t('api.error.<CODE>.title')
   → Jika ketemu → gunakannya sebagai judul toast (reaktif ikut bahasa)
2. Baca error.errors (422 validation) atau error.details.fields
   → Gunakan pesan field pertama sebagai detail toast
3. Fallback berdasarkan HTTP status (401, 403, 409)
```

Karena judul menggunakan `i18n`, toast langsung reaktif saat bahasa diganti via Topbar.

## Best Practice Implementasi Komponen

1. Pakai key semantik, hindari key berupa kalimat panjang.
2. Tambah key EN+ID bersamaan dalam PR yang sama.
3. Untuk enum/status, gunakan mapping i18n (jangan tampilkan raw code):
   - `active/inactive`
   - `no_account/invitation_pending/account_active`
   - `owner/admin/member`
4. Untuk breadcrumb/title, gunakan key modul (`tenant.errors.forbidden.page_title`), bukan string literal.
5. Untuk interpolasi, gunakan placeholder:
   - `tenant.dashboard.plan_badge: "Plan: {{plan}}"`
6. Data navigasi shell tidak boleh membawa label user-facing hardcoded; gunakan translation key seperti `layout.shell.nav.items.settings`.
7. **Semua `catch` block di modal/form** wajib menggunakan `parseApiError` + `notify` bukan `alert` atau string hardcode.

## Checklist PR i18n

1. Tidak ada string user-facing hardcoded di file yang diubah.
2. Key EN dan ID lengkap (tidak boleh salah satu saja).
3. `npm run lint` hijau.
4. `npm run typecheck` hijau.
5. `npm run build` hijau.
6. Uji manual switch bahasa di modul yang terdampak.
7. Uji error path API memastikan message/hint ikut locale aktif.

## Cakupan Runtime Prioritas

Prioritas i18n wajib penuh:

- Auth
- Admin dashboard/tenants/subscriptions
- Tenant modules: dashboard, members, roles, invitations, whatsapp
- Tenant Master Data: categories, tags, currencies, uom
- Tenant settings: profile, branding, localization, billing
- Tenant error pages: forbidden, upgrade-required, access-required
- Shared shell chrome: sidebar, topbar, customizer, nav search

## Anti-Pattern yang Dilarang

- Menaruh semua key di satu file `tenant.json`.
- Menampilkan `error.code` mentah ke end-user sebagai pesan utama.
- Menerjemahkan hanya frontend, tapi backend tetap English hardcoded.
- Menambah bahasa baru tanpa sinkronisasi header + middleware + fallback policy.
- Hardcoding string error langsung di `notify.error('...')` tanpa `parseApiError`.
- Menambah locale key hanya di EN tapi tidak di ID (atau sebaliknya).
